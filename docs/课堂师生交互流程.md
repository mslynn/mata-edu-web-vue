# 课堂师生交互流程

## 1. 文档目的

本文用于说明老师端课堂与学生端课堂之间的实时交互流程，以及 WebSocket JSON 消息中 `type`、`action` 的含义，方便前后端统一联调。

当前课堂交互主要基于 WebSocket 通知完成，老师端发消息，学生端接收后更新课堂状态、资源弹窗、任务弹窗等界面。

---

## 2. 消息总览

课堂中目前主要使用以下消息类型：

| `type` | 说明 |
| --- | --- |
| `CLASS_BEGIN` | 开始上课 |
| `CLASS_END` | 结束上课 |
| `SCREEN_SHARE_START` | 开始屏幕共享 |
| `SCREEN_SHARE_STOP` | 停止屏幕共享 |
| `COURSEWARE_SEND` | 下发课程资源 |
| `COURSEWARE_REVOKE` | 撤回课程资源 |
| `CLASS_INTERACTION` | 课堂互动控制消息，具体动作由 `action` 决定 |
| `KICKOUT` | 老师将学生移出课堂 |

说明：

- `CLASS_INTERACTION` 是一个总线型消息。
- 具体是“全屏控制”“下发任务”“撤回任务”“打回重做”，由 `action` 字段区分。

---

## 3. `CLASS_INTERACTION.action` 含义

当前课堂中已使用的 `action` 含义如下：

| `action` | 说明 | 学生端行为 |
| --- | --- | --- |
| `fullscreen_enter` | 老师开启全屏管控 | 学生端弹出全屏提示并进入全屏流程 |
| `fullscreen_exit` | 老师关闭全屏管控 | 学生端退出全屏 |
| `question` | 老师下发课堂任务 | 学生端刷新当前课堂任务，并弹出课堂任务窗口 |
| `question_revoke` | 老师撤回课堂任务 | 学生端刷新当前课堂任务；若无有效任务则关闭任务弹窗 |
| `question_redo` | 老师打回重做 | 学生端刷新课堂任务；若任务弹窗已打开则同步刷新左侧任务列表 |

说明：

- `question`、`question_revoke`、`question_redo` 都属于“课堂任务相关动作”。
- 这三类动作必须带上 `chapterId`，学生端才能按正确章节查询对应任务。

---

## 4. 公共字段说明

课堂消息中常见字段说明如下：

| 字段 | 说明 |
| --- | --- |
| `type` | 消息主类型 |
| `action` | `CLASS_INTERACTION` 下的具体动作 |
| `peerId` | 当前课堂班级标识，课堂互动消息里通常使用班级 ID |
| `classId` | 班级 ID |
| `courseId` | 课程 ID |
| `chapterId` | 章节 ID，课堂任务查询必须依赖该字段 |
| `chapterName` | 章节名称 |
| `className` | 班级或课堂名称 |
| `teacherPeerId` | 老师屏幕共享相关的 PeerJS ID |
| `resourceId` | 资源 ID / 任务资源 ID |
| `taskId` | 学生任务 ID 或任务记录 ID |
| `taskName` | 任务名称 |
| `resourceCategory` | 资源分类，如随堂练习、自定义练习题、学生编程任务等 |
| `distributeType` | 下发类型，`1` 表示学生端，`2` 表示小组端 |
| `coursewareList` | 课件列表 |

---

## 5. 课堂主流程

### 5.1 开始上课

老师端开始上课后，向学生端广播：

```json
{
  "type": "CLASS_BEGIN",
  "classId": "1996788444002480128",
  "className": "一年级 1 班",
  "courseId": "10001",
  "chapterId": "20001",
  "chapterName": "Lesson 1",
  "peerId": "teacher_peer_xxx"
}
```

学生端收到后：

1. 更新当前课堂上下文。
2. 进入学生课堂页。
3. 初始化屏幕共享连接。
4. 保存“回到课堂”缓存。

---

### 5.2 下发课程资源

老师端发送课件：

```json
{
  "type": "COURSEWARE_SEND",
  "peerId": "1996788444002480128",
  "classId": "1996788444002480128",
  "courseId": "10001",
  "chapterId": "20001",
  "coursewareList": [
    {
      "resourceId": "30001",
      "fileName": "Teacher Guide.pdf",
      "resourceUrl": "https://xxx/teacher-guide.pdf"
    }
  ]
}
```

学生端收到后：

1. 更新当前课堂上下文。
2. 刷新课件列表。
3. 自动弹出课程资源窗口。

---

### 5.3 下发课堂任务

老师端下发任务时，发送：

```json
{
  "type": "CLASS_INTERACTION",
  "peerId": "1996788444002480128",
  "action": "question",
  "classId": "1996788444002480128",
  "courseId": "10001",
  "chapterId": "20001",
  "taskId": "40001",
  "resourceId": "50001",
  "taskName": "随堂练习题1",
  "resourceCategory": 6,
  "distributeType": 1
}
```

学生端收到后：

1. 更新当前课堂上下文，重点是 `chapterId`。
2. 按当前 `chapterId` 查询学生任务列表。
3. 弹出课堂任务窗口。
4. 课堂任务窗口展示当前课堂下发的任务。

说明：

- 这里必须带 `chapterId`。
- 如果没有 `chapterId`，学生端只能查到错误章节或查不到任务。

---

### 5.4 撤回课堂任务

老师端撤回任务时，发送：

```json
{
  "type": "CLASS_INTERACTION",
  "peerId": "1996788444002480128",
  "action": "question_revoke",
  "classId": "1996788444002480128",
  "courseId": "10001",
  "chapterId": "20001",
  "taskId": "40001",
  "resourceId": "50001",
  "taskName": "随堂练习题1",
  "resourceCategory": 6,
  "distributeType": 1
}
```

学生端收到后：

1. 更新当前课堂上下文。
2. 重新查询该章节课堂任务。
3. 如果当前课堂已经没有有效任务，则关闭任务弹窗或显示空状态。

---

### 5.5 打回重做

老师端打回重做时，发送：

```json
{
  "type": "CLASS_INTERACTION",
  "peerId": "1996788444002480128",
  "action": "question_redo",
  "classId": "1996788444002480128",
  "courseId": "10001",
  "chapterId": "20001",
  "taskId": "40001",
  "resourceId": "50001",
  "taskName": "随堂练习题1",
  "resourceCategory": 6,
  "distributeType": 1
}
```

学生端收到后：

1. 更新当前课堂上下文。
2. 刷新课堂任务数据。
3. 如果课堂任务弹窗已打开，则同步刷新左侧任务列表与当前任务状态。

---

### 5.6 全屏控制

老师端开启全屏：

```json
{
  "type": "CLASS_INTERACTION",
  "peerId": "1996788444002480128",
  "action": "fullscreen_enter"
}
```

老师端关闭全屏：

```json
{
  "type": "CLASS_INTERACTION",
  "peerId": "1996788444002480128",
  "action": "fullscreen_exit"
}
```

---

### 5.7 结束上课

老师端结束课堂：

```json
{
  "type": "CLASS_END",
  "classId": "1996788444002480128"
}
```

学生端收到后：

1. 清理课堂缓存。
2. 清理计时器。
3. 关闭课堂态连接。
4. 显示下课提示。

---

## 6. 联调注意事项

### 6.1 课堂任务相关消息必须带 `chapterId`

这是最重要的一条：

- 学生端课堂任务查询依赖 `chapterId`。
- 老师端发送 `question`、`question_revoke`、`question_redo` 时，如果不带 `chapterId`，学生端就无法准确查到当前课堂对应任务。

### 6.2 `peerId` 在课堂互动消息中建议统一表示班级 ID

当前实现里，`CLASS_INTERACTION` 的 `peerId` 更接近课堂班级标识，不建议和 `teacherPeerId` 混用。

建议约定：

- `peerId`：班级 ID / 课堂房间标识
- `teacherPeerId`：老师屏幕共享 PeerJS ID

### 6.3 学生端展示的是“课堂当前下发任务”

课堂任务弹窗的目标不是展示整章历史任务，而是展示当前课堂正在下发中的任务。

因此建议后端或消息层保证：

1. 下发时明确告诉学生端当前任务属于哪个章节。
2. 撤回时明确告诉学生端撤回的是哪个任务。
3. 学生端收到后按当前课堂上下文刷新，不混入无关章节数据。

---

## 7. 推荐联调顺序

建议按以下顺序联调：

1. `CLASS_BEGIN`
2. `COURSEWARE_SEND` / `COURSEWARE_REVOKE`
3. `CLASS_INTERACTION + question`
4. `CLASS_INTERACTION + question_revoke`
5. `CLASS_INTERACTION + question_redo`
6. `CLASS_INTERACTION + fullscreen_enter / fullscreen_exit`
7. `CLASS_END`

---

## 8. 当前已约定的任务动作

当前课堂任务动作统一如下：

```text
question         = 下发课堂任务
question_revoke  = 撤回课堂任务
question_redo    = 打回重做
```

如果后续后端需要统一命名，可以在前后端同步替换，但同一阶段内建议保持不变。
